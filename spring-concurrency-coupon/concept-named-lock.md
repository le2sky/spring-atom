### 네임드 락

- GET_LOCK() 함수를 이용해 임의의 문자열에 대해 잠금을 설정할 수 있다.
- 대상이 테이블이나 레코드 또는 AUTO_INCREMENT와 같은 데이터베이스 객체를 잠그는 것이 아니다.
- 네임드락은 단순히 사용자가 지정한 문자열에 대해 획득하고 반납한다.
- 여러 클라이언트가 상호 동기화를 처리해야 할 때 네임드 락을 이용하면 쉽게 해결할 수 있다.
- 자주 사용되지는 않지만, 웹 서버가 어떤 정보를 동기화해야 하는 요건처럼 클라이언트가 상호 동기화를 처리해야할 때 사용된다.
- 많은 레코드에 대해 복잡한 요건으로 레코드를 변경하는 트랜잭션에 유용하게 사용될 수 있다.
- 동일 데이터를 변경하거나 참조하는 프로그램끼리 네임드 락을 걸고 쿼리를 실행하면 해결 가능함(배치 프로그램에서 데드락을 막기 위해서)
- GET_LCOK()을 이용해 획득한 잠금은 트랜잭션이 종료되어도 해제되지 않는다.

```mysql
select get_lock('mylock', 2);
select is_free_lock('mylock');
select is_used_lock('mylock');
select release_lock('mylock');
select release_all_locks();
```
